# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

- リモートリポジトリ：ネット上に配置して複数人で共有するためのリポジトリ
- ローカルリポジトリ：開発者一人一人が使用する為に自分のPC上に配置するためのリポジトリ


## プッシュとマージの違いは何でしょう？

- プッシュ：ローカルリポジトリの変更をリモートリポジトリに反映させること
- マージ：ローカルリポジトリで別のブランチの作業内容をブランチに取り込むこと


## コミットとプッシュの違い

- コミット：ローカルリポジトリで変更履歴を残すこと
- プッシュ：ローカルリポジトリの変更履歴をリモートリポジトリに反映させること


## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

- 一目で変更内容がわかるものにする


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

- ローカルでマージするフロー：ローカルリポジトリで変更を直接反映させるフロー
- プルリクエストでマージするフロー：ローカルリポジトリで行なった変更をリモートリポジトリにプッシュした後に他の開発者にレビューを依頼して承認を得てからマージするフロー


## コンフリクトを起こしてしまった場合、どう対処するべきですか？

- 各チームのルールに合わせて、どのブランチをもとにして新しくブランチを切るかを正確に確認して、変更を取り込む
